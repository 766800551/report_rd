        # 简介

        • 通过 wps 加载项来实现报表的生成，不再分发在线版，后续都是离线版。
        • 预留了自动备份功能，但是暂不开启。
        • 此次更新日期为：2020-11-29，下载地址 (opens new window)

        # 它是如何生成的？

        报表生成插件主要是读取已经被激活的工作表，然后根据 report 设置（主要是报表的基本参数），section 设置（每一个数据块的范围），field 设置（节点所拥有的属性）来对表格的所有属性进行读取，最后进行 xml 拼接，生成的 xml 直接拷贝即可使用。后续会逐步实现将配置的 xml 一键转换成为旧版本（5.1 以下的）以及预览

        # 它为什么不支持版本为 5.1 以下的系统？

        5.1 以下的版本存在很多问题，xml 报表属性的不统一，没有勾选框节点，双页设置方式不统一等等，这些问题会导致如果兼容 5.1 以下的，那么这个插件的复杂性要上升很多。

        # 5.1 版本以下的系统是否能够使用？

        5.1 以下的系统是能够使用的，但是像一些特殊的标签，那么就可能存在不支持的情况，另外后续会考虑将生成的 xml 进行转换，转换成 5.1 以下的系统能够使用的。。。

          新增数据如下图：

          多个节点呢？修改？是否只能是数字？

          下图是对其操作的一个演示。

          • 多个节点就多个大括号
          • 如果大括号不是闭合的话，不会识别
          • 可以随意加文字或符号，但是文字和符号会被去除，并不做任何保留
          • 如果是节点的话，大括号里面必须要有数字

          这里说下原理，当然你可以选择跳过。

          当你录入内容后，wps 会监测到单元格发生改变，这个时候就会判断内容，是否满足节点，是什么样的操作（新增还是删除还是修改），然后当你鼠标离开这个单元格后这个单元格改变的事件算是触发完成，这个时候相应数据会写进 localstorage，当你再次点击这个单元格的时候，会触发单元格被选中的事件，这个时候，会将相应数据进行展示。

          注意：由于大括号里面必须要有数字，但是 docNode.Time 又是特殊的节点，所以表格里面需要写 99999。

        # report设置

        # 纸张宽度

        文书宽度，需要为大于0的整数

        # 纸张高度

        文书高度，需要为大于0的整数

        # 外边距

        文书的外边距，需要用英文逗号隔开，分别是左，右，上，下，需要为大于0的整数。

        # 横向打印

        默认为否，如果选是则横着打印，否则竖着打印。

        # 行高缩放

        指excel的行高按照什么比例缩放到xml报表中，大于0的小数。

        # 双页

        默认不启用，如果是双页文书则需要启用

        # A页范围

        指定excel的一个范围来生成文书，如果是单页的话，只需要设置A页范围，如果是双页则需要设置B页范围

        # B页范围

        指定excel的一个范围来生成文书，需要双页配置为已启用

        # 保存/重新载入字体

        当配置发生变化时，需要点击保存才会生效（只是针对report设置）。

        因为wps遍历字体并重新编号的速度并不可观，因此当excel发生字体变化时，请点击这个按钮，否则发生变化的字体便是未定义。

        # 导出全部配置

        将所有报表生成的配置进行导出，导出的是一个二进制文件

        # 导入全部配置

        将所有报表生成的配置进行导入，导入的是一个二进制文件

        # 清除全部缓存

        将所有的配置进行清空（不会清除默认配置，但是会还原）

        # section设置

        每一行的section都对应着xml里面的一对section标签。

        # 报表块类型

        对应着section标签里面的kind，区段类型。

        # 范围

        指一个section在表格里面所作用的范围是多大，选中范围内单元格，点击即可进行录入。

        # 列数

        VerticalDetail专有，指定列数（后续可能会考虑删除此参数，因为是多余配置，主要因为当指定范围后，其实表格里面的列数就已经明确了）

        # 索引、合并索引

        暂未用到，预留的双页参数。

        # 清除section缓存

        点击后会删除目前所有的section配置并且进行刷新。

        # field设置

        field为节点的配置，相关配置会写入LocalStorage里。

        清除field缓存在存在类型为opfield的时候会误删，所以请谨慎使用，后续会修复这个bug

        # 参数

        内容有点多，暂略，后续有空补上，可以参考报表的配置文档。

        # 按钮

        # 高亮节点/取消高亮

        激活后，会标识那些单元格已经存在节点。

        # 选框替换

        对选中的单元格进行添加replace="1,√"这样的配置。

        # 签名

        对选中的单元格进行添加format="signature"这样的配置

        # 日期时间

        对选中的单元格进行添加format="yyyy-MM-dd HH:mm" dataType="System.DateTime"这样的配置

        # 日期

        对选中的单元格进行添加format="yyyy-MM-dd" dataType="System.DateTime"这样的配置

        # 时间

        对选中的单元格进行添加format="HH:mm" dataType="System.DateTime"这样的配置

        # 清除field缓存

        目前存在bug，慎用（最好不用）

        # 排序方式

        生成xml的时候可以选择排序方式，5.1以下的版本请慎重选择，因为低版本的可能不支持docNode.Time参数

        排序方式默认是以时间正式排序，如果是有出入量统计的护理记录单的话，可以将出入量统计时间也进行选中。

        # 页脚

        页脚默认选中生成，如果不需要可以取消勾选，取消勾选则不生成。
